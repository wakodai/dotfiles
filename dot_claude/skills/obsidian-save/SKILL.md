## 概要

このスキルは、会話中で行った調査・議論・検証の内容を整理し、Obsidianの保管庫にMarkdownファイルとして保存する。

## 保存先

```
/Users/wakodai/Library/Mobile Documents/iCloud~md~obsidian/Documents/obsidian/
```

## ファイル名

- 内容のトピックに基づいて日本語で自動生成する
- Obsidianで扱いやすいよう、スペースは許容するがファイルシステムで問題になる文字（`/`、`:`、`*`、`?`、`"`、`<`、`>`、`|`）は避ける
- 例: `同一端末への2つのIPアドレスとTailscale経由のファイル転送調査.md`

## フロントマター

以下の形式で必ず付与する:

```yaml
---
tags:
  - tag1
  - tag2
created: YYYY-MM-DD HH:mm
---
```

### tags

- 会話の内容から適切なタグを自動判断して付与する
- 技術的なトピックならツール名・プロトコル名・分野名など（例: `network`, `tailscale`, `vpn`, `aws`, `docker`）
- 非技術的なトピックでも適切な分類タグを付ける
- 2〜5個程度が目安

### created

- `YYYY-MM-DD HH:mm` 形式で保存時点の日時を記録する
- 現在時刻を取得して埋める（`date "+%Y-%m-%d %H:%M"` などで取得）

## 本文の構成

会話の内容を、後から読み返して理解できる独立したドキュメントとして構成する。会話の流れをそのまま転記するのではなく、調査結果として再構成する。

### 構成の指針

1. **見出し（H1）** — トピック全体を表すタイトル
2. **背景・前提** — 何を調べたのか、なぜ調べたのか（会話の文脈から読み取れる範囲で）
3. **調査内容** — 調べた結果、検証した結果を整理して記載。表・コードブロック・箇条書きを活用して読みやすくする
4. **まとめ** — 結論や要点を簡潔に

### 注意点

- 会話中で実行したコマンドの出力結果など、具体的なエビデンスは適宜含める
- コードブロックにはシンタックスハイライト用の言語指定を付ける
- Obsidianで正しく表示されるMarkdown記法を使う（Callout記法なども活用してよい）
- 冗長にならないよう、要点を絞って記述する

---

## コンテキスト
- 現在のブランチ: !git branch --show-current
- 変更されたファイル一覧: !git status --short
- 変更差分: !git diff

## タスク

1. 差分が存在しない場合はコミットせず、その旨を伝えて終了する
2. `git add -u` で modified のファイルをすべてステージングする
3. 上記の差分内容を分析し、Conventional Commits 形式でコミットメッセージを **3案** 作成する
   - 形式: `type(scope): 概要`
   - type: feat / fix / refactor / docs / style / test / chore から適切なものを選ぶ
   - scope: 変更の主な対象（省略可）
   - 概要: 日本語で簡潔に（50文字以内目安）
   - 3案はそれぞれ粒度や観点を変えること（例: 具体的な変更内容 / 変更の目的 / 影響範囲）
   - 必要に応じて本文（body）も付ける
4. 以下のフォーマットで提示し、ユーザーに選択を求める：

```
## コミットメッセージ案

1️⃣ fix(auth): ログイン時のトークン検証ロジックを修正
2️⃣ fix: 認証エラーで不正なセッションが残る問題を解消
3️⃣ fix(auth): JWTの有効期限チェックが無視されていた不具合を修正

番号を選んでください（1/2/3）。
または修正指示や独自のメッセージを入力してもOKです。
```

5. ユーザーの回答に応じて対応する：
   - **番号（1/2/3）を選択** → そのメッセージで `git commit` を実行
   - **修正指示やフィードバック** → 指示を反映した新しい3案を再提示する
   - **独自のメッセージをそのまま入力** → そのメッセージで `git commit` を実行
6. コミット完了後、コミットハッシュを表示する

## 注意事項
- untracked（新規）ファイルは追加しない（`git add -u` を使う）
- $ARGUMENTS が指定された場合は、それをコミットメッセージのヒントとして考慮する
- コミット実行前に必ずユーザーの選択を待つこと。勝手にコミットしない
